:::note

Though it is possible to send transactions with iota.rs, we strongly recommend that you use official
[`wallet.rs`](https://wiki.iota.org/wallet.rs/welcome) library together with the
[`stronghold.rs`](https://wiki.iota.org/stronghold.rs/welcome) enclave for value-based transfers. This combination
incorporates the best security practices while dealing with seeds, related addresses and `UTXO`.

:::

## Installing the Library

### Requirements

You can find all the requirements for using the iota.rs library in the
[Getting Started](getting_started.md#requirements) section.

### Clone the Repository

To install the library, you will first need to clone the repository by running the following command:

```bash
git clone https://github.com/iotaledger/iota.rs
```

### Create Your Configuration File

Once you have [cloned the repository](#clone-the-repository), you should change directory into the projects by running:

```bash
cd iota.rs
```

Once you are in the project directory you should make a copy of the
[.env.example file](https://github.com/iotaledger/iota.rs/blob/dev/.env.example) and rename it as `.env`. You can do
that with your file manager or IDE of choice, or by running the following command:

```bash
cp .env.example .env
```

## Running Examples

Once you have [cloned the repository](#clone-the-repository), and
[created your configuration file](#create-your-configuration-file) you are ready to run examples.  You can use the
following command to so replacing `01_get_info` for your example of choice from the
[examples](https://github.com/iotaledger/iota.rs/tree/dev/examples) directory.

```bash
cargo run --example 01_get_info --release
```
